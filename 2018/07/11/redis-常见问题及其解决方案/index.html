redis-常见问题及其解决方案

2018-07-11 PV:

redis 客户端 不能存储问题

(error) MISCONF Redis is configured to save RDB snapshots, but it is currently not able to persist on disk. Commands that may modify the data set are disabled, because this instance is configured to report errors during writes if RDB snapshotting fails (stop-writes-on-bgsave-error option). Please check the Redis logs for details about the RDB error.

– 原因：

强制关闭Redis快照导致不能持久化。

解决方案：

运行config set stop-writes-on-bgsave-error no命令后，关闭配置项stop-writes-on-bgsave-error解决该问题。

root@ubuntu:/usr/local/redis/bin# ./redis-cli 127.0.0.1:6379> config set stop-writes-on-bgsave-error no OK 127.0.0.1:6379> lpush myColour "red" (integer) 1

关于java端连不上redis pool的问题（ 逐一排除 ）

  1. 确认java程序无误

  2. 可能是redis配置文件中端口的限制

    进入redis.conf文件 默认配置是本地IP 所以直接将里面的bind 127.0.0.1注释掉

  3. 防火墙问题

    1. 首先是虚拟机防火墙问题命令窗口将redis 6379 端口放开
      命令：`/sbin/iptables -I INPUT -p tcp –dport 6379 -j ACCEPT
      /etc/rc.d/init.d/iptables save`
    2. 如果仍然不能连接，则断开防火墙（有风险）
      1. 永久性生效，重启后不会复原
        开启： chkconfig iptables on
        关闭： chkconfig iptables off
        2.即时生效，重启后复原
        开启： service iptables start
        关闭： service iptables stop

  4. 考虑redis版本或是java 程序的两个jar包 的不兼容的问题，更换匹配的jar包

  5. 注意：在更换redis版本前
    1. 首先要关闭客户端，执行命令./redis-cli shutdown或者干掉线程，命令ps -ef|grep redis进行查看线程，执行命令kill -9 线程号
    2. 其次要清空虚拟机中的所有redis文件再重新安装

经过这几个步骤基本可以解决问题啦~~~
